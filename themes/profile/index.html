Axiom 01 Core Truths
Rules to follow when desining and coding User Interfaces

Axiom. Rules that set you free. The only CSS Framework with rules to tame the madness. Most CSS frameworks contains tools, but no direction or rules on how and how not to use them. They follow no standard axioms for design and layout. Axioms are things that are presumed to be tree, with which other things can be derived or determien through logic. Axiom, is a framework that provides solid rules and design direction to create a beautiful free experience with extremely easy to understand clean code.

  • No rules are absolute (haha!) - Don't make stupid decisions or work arounds if you need to do something or need a one off or edge case just to not violate these rules, these should be treated as strict guideliens that should only be broken is there is a solid need. When in doubt, follow all of them to the T and you'll design much fster with a higher quailty and consistency rate (and way more enjoyable1).
  • Elements can contain like elements or any element father down the Axiom Element Heirarchy Tree.
  • Behaviors should dictate class names
  • Minimalism Additions to should be added for custimizations, not overriding or changing values.
  • Default classes and styles should work directly on the elements and not require presentational classes
  • If html can't be changed, then the class "section" or "article" or the geneihttps://sol-roth-media.github.io/axiom/c content should be applied to all content.
    • \
  • All buttons are assumed to be primary, no designation for default. No extra code for defaults. Everything is relative to the tree default.
  • All like elements (structure) should have the EXACT same horizontal and vertical alignment. Extra classes should be used to adjust the default alignment. (divs, articles, etc etc)

Hierachy and Element Specific Rules

  • Dividers (Layout and programatic sectioning)
    • Dividers should not directly contain content, they should contain other dividers or containers. They should be grid, flexbox or block display.
        • Sample Element List
      • < div >
      • < main >
      • < footer >
      • < nav >
      • < header >
  • Containers (Large Content and content sectioning)
    • All containers should be block display, or inline-block display. When possible content sectioning should be used for containers and all related styles should use the root content element
        • Sample Element List content sectioning elements - non-exhaustive list
      • < address >
      • < article >
      • < aside >
      • < section >
      • < form >
  • Wrappers (individual disable items - content, style and misc)
    • All wrappers should be inline display or no display property (default).
        • Sample Element List
      • < span >
  • Content Everything that's not a divider, wrapper or container.
      Sample Element List
    • < table >
    • < p >
    • < a >
    • < h1 > - < h6 >

Color Scheme and Pallet

Single color full scheme generation - Axiom uses math to calualte matching hues and brightness / darkness to ensure readability and aesthetically matching color combinations. You only need to input a single color (you can custimize if you wish) and axiom does the rest for you. Never worry about basic color combinations again. Using a simple naming structure, "container" and "on-container", "primary", "on-primary", it's easy to always know what color whould be matched with what background color and what a container should be colored that contains other colored elements. Make color EASY!
